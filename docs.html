Getting Started

Welcome to LadyJek's delivery API. The LadyJek API is organized around REST. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors.

For further inquiries, please do not hesitate to contact us at dev@ladyjek.com

Authentication

Authenticate your account when using the API by including your secret API key / client ID in your POST request. Your API keys carry many privileges, so be sure to keep them secret! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

Setting-up your order form

In order to successfully submit your delivery requests, LadyJek API requires a specific structure along with the necessary input items for each order.

The necessary data fields that have to be present in the form are as follows:

  • From:
    • Full Name
    • Contact No.
    • Address (including zip code)
  • To:
    • Full Name
    • Contact No.
    • Address (including zip code)
  • Item Description

Add an order

Once our form is ready, adding an order is just a simple POST request away. Using your preferred method, assign a button that will generate an HTTP POST request to our API endpoint.

The following example will use AJAX to generate a POST request to http://www.lady-delivery.firebaseio.com/order.json

		
// variable declarations
...

// JSON data
...

// POST ajax

$.ajax({
    type: 'POST',
    url: 'https://lady-delivery.firebaseio.com/order.json',
    contentType: "application/json; charset=utf-8",
    data: JSON.stringify(newOrder),
    success: function(data) {
        console.log("Order added!", data);
    },
    error: function (jqXHR) {
        if (jqXHR.status == 401) {
            alert("401: Authentication Error!");
        } else if (jqXHR.status == 400) {
            alert("400: Invalid data format in input");
        };
    }
});
Errors

LadyJek uses conventional HTTP response codes to indicate the success or failure of an API request. In general, codes in the 2xx range indicate success, codes in the 4xx range indicate an error that failed given the information provided, and codes in the 5xx range indicate an error with LadyJek's servers (these are rare).

Not all errors map cleanly onto HTTP response codes, however. When a request is valid but does not complete successfully, we return a 402 error code.


HTTP status code summary

Code Name Description
200 OK Everything worked as expected
400 Bad Request The request was unacceptable, often due to missing a required parameter.
401 Unauthorized No valid API key provided.
402 Request Failed The parameters were valid but the request failed.
404 Not Found The requested resource doesn't exist.
409 Conflict The request conflicts with another request (perhaps due to error in assigning order ID).
429 Too Many Requests Too many requests hit the API too quickly.
500, 502, 503, 504 Server Errors Oops, something went wrong on LadyJek's end. Let us know!
