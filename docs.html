Getting Started

These are the steps to implement LadyJek Delivery API in your system:

  1. Get in touch with us to confirm our partnership here
  2. Receive your client ID in order to pass our authentication gateway for your orders
  3. Develop your order form and link it with our sandbox API endpoints to start testing
  4. Move on to the launch phase and ship! 🚀
Authentication

Authenticate your account when using the API by including your secret API key / client ID in your POST request. Your API keys carry many privileges, so be sure to keep them secret! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

* for the current being, your client ID will be sent to you through an authorization email.

Setting-up your order form

In order to successfully submit your delivery requests, LadyJek API requires a specific structure along with the necessary input items for each order.

The necessary data fields that have to be present in the form are as follows:

  • From:
    • Full Name
    • Contact No.
    • Address (including zip code)
  • To:
    • Full Name
    • Contact No.
    • Address (including zip code)
  • Item Description

Sandbox

The LadyJek Delivery API Sandbox is for you to test the functionality of your application without making legitimate calls to our main delivery platform. All request to the sandbox API endpoints will be ephemeral.

The requests to the sandbox environment can be sent to this url:

https://lady-delivery-sandbox.firebaseio.com/order.json

When you have successfully added an order, you will be able to retrieve your order in a JSON format at the following url:

https://lady-delivery.firebaseio.com/order/{order_id}

The same method can also be applied to your sandbox environment.

https://lady-delivery-sandbox.firebaseio.com/order/{order_id}
Errors

LadyJek uses conventional HTTP response codes to indicate the success or failure of an API request. In general, codes in the 2xx range indicate success, codes in the 4xx range indicate an error that failed given the information provided, and codes in the 5xx range indicate an error with LadyJek's servers (these are rare).

Not all errors map cleanly onto HTTP response codes, however. When a request is valid but does not complete successfully, we return a 402 error code.


HTTP status code summary

Code Name Description
200 OK Everything worked as expected
400 Bad Request The request was unacceptable, often due to missing a required parameter.
401 Unauthorized No valid API key provided.
402 Request Failed The parameters were valid but the request failed.
404 Not Found The requested resource doesn't exist.
409 Conflict The request conflicts with another request (perhaps due to error in assigning order ID).
429 Too Many Requests Too many requests hit the API too quickly.
500, 502, 503, 504 Server Errors Oops, something went wrong on LadyJek's end. Let us know!
Check order status

LadyJek Delivery API currently provides three stages for the information on delivery progress. The three stages are:

  • posted : The order is successfully received by LadyJek as a delivery request. The default value is true
  • picked : LadyJek biker has picked up the item from the trip origin. The default value is false
  • delivered : The item has been successfully delivered to the trip destination. The default value is false

Checking the status of a specific order is just a simple GET request to the following url:

https://lady-delivery.firebaseio.com/order/{order_id}/status.json

If the order_id is valid, this action will give you the following result in JSON format:

		
{
  delivered: {
    active: false,
    time: ""
  },
  picked: {
    active: false,
    time: ""
  },
  posted: {
    active: true,
    time: "17:14" // timestamp when status is true
  }
}
Update delivery details

The function to update details of an order is still currently under development.

  • Please contact our customer service directly to change the information by providing your name, company name, and order_id.
  • You can only change delivery details when the item has not yet been picked-up by a LadyJek Biker.
  • When an order is picked-up, the trip will be billed in full.
Cancel an order

The function to cancel an order is still currently under development.

  • Please contact our customer service directly to cancel an order by providing your name, company name, and order_id.
  • Since the delivery process will immediately begin when an order is posted, order cancellation will be free-of-charge for 10 minutes after the posted time.
  • After 10 minutes, there will be a cancellation fee of half of the total trip value.
  • When an order is picked-up, the trip will be billed in full.
